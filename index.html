04、执行意图：大脑服从「自动程序」，对抗拖延症

Posted on 2019-05-07 | Visitors:
Words count in article: 2.3k | Reading time ≈ 7

每个人都可以雄心壮志，只有赢家才能付诸实行。

不过只是上传音频、修全文稿、联络不同老师等等的庶务…我都常常犯拖延症，无限接近ddl才开工。

但邓亚萍面对的任务，是穿着20斤的沙衣、戴着5斤的负重带，每天持续挥拍上千下；从26个字母都认不全，到成为剑桥博士学霸。

今天想问她：面对让人烦躁的琐事，世界冠军怎么坚持做下去？

早上好，我是邓亚萍，你的冠军心态教练。今天，你准备好成为更不拖延的自己了吗？

接下来这节课，我要先跟你说一说我的经验。

我要告诉你，我是如何在退役以后，从26个字母都认不全，到后来英文程度好到，可以让我拿到剑桥大学的博士。

更重要的，我会帮你从我的经验中，提炼出一个专业心理学的小技巧，让你能像控制电脑程序一样，控制你自己，自动自发去做一件事。

啥意思？简单来说，就是让你在不想读书学习的时候，还能自动自发去读书学习，让你在不想工作的时候呢，也能够自动自发去工作。

这个效果，听上去是不是很神奇？其实啊，大多数的运动员，尤其是顶尖运动员，都有这个能力。

你肯定看过一种报导，说咱们运动员，尤其是最顶尖的运动员哈，训练都特别刻苦。这话呢说得没错。

但是呢，这种报导通常也会说，运动员有多热爱各种训练，这个话呢，就不完全对了。

就我所知，很多优秀的运动员，训练都很勤快，但你问起他，你喜欢训练吗？他也会说：「训练有时候也挺烦的」。

这是不是跟你想象的不一样？我说的更直接一点，基本上，顶尖运动员都掌握了一种小方法，就算我们并不特别喜欢做一件事，觉得这件事枯燥乏味，我们还是能坚持去做。

我举我的故事给你听哈！当然，我说我自己不喜欢训练，你肯定不信。所以我今天，是要跟你讲讲我学英文的故事。

我24岁退役后，被当时的奥运委员会主席，萨马兰奇先生任命为国际奥委会的委员，但问题来了，奥委会委员，是要说英文的啊！这代表我必须要去学英文啊！

可是我打了一辈子的乒乓球，根本没啥时间学习，英文当然是不好。你知道我的程度多差吗？我就连26个英文字母，都认不全！

在这种状况下，我要把英文学好，真的要下很大的苦功夫。

我给自己定了计划，一切从头开始，从第一个字母、第一个单词背起，一天保证14个小时的学习时间，每天早上5点准时起床，读音标、背单词、练听力，晚上呢？就整理讲义，温习功课，直到深夜12点。

很吓人，对吧？一开始，我还能靠打鸡血强行支撑，但我很快就发现，人的意志力是有限的，我也会有倦怠，甚至想拖延的时候。

比如说我有时候学习得太晚，睡得不好，我也会想要赖床，打开书本后，我偶儿也会觉得不想学习，不想背单字，想去做其他事。

我很诚实地说，跟你一样，我当然也不喜欢背单字、我当然也觉得每天做同样的事很枯燥。

但是，我还是坚持了我的读书计划，每天学习英文14个小时，持续了三年。

我是怎么办到的？简单说，我是把「背单字」这个事，想象成我在训练场练球。

说的更具体一点，我是清楚地告诉自己： 就像到了训练场，就马上要练正手击球500球，我一坐到书桌前，就是要先背20个单字！

就这么简单！

一但我不断的告诉自己，到书桌前，不管怎样就是要背20个单字，并且一次又一次实践，到了后来，如果我到了书桌前却不背单字，我还觉得难受。

就凭我这种「条件式反应」，三年后，我在清华的毕业典礼上，可是用全英文跟老师致辞的呢！

学英文的人那么多，但我能成功，是因为什么？我说给你听。

一般人呢，在做计划的时候，都不会考虑到各种临时的状况，也通常都会高估自己的意志力。

就比如学英文，很多人的目标都是每天背100个单字，但大部分的人，到了最后，也会去找各式各样、五花八门的理由，说服自己今天可以自己放一天假，不用那么辛苦。

拖延了一次，就会继续拖延第二次…

但是，为啥顶尖运动员，通常没有这种问题呢？为啥我们即使不喜欢训练，我们还是可以坚持持续训练好几年呢？

这是因为，这些运动员已经锻炼好自己的大脑。面对枯燥的训练，我们是用「条件反射」去督促自己去锻炼。

就拿我为例，以前我每天去晨练，都要练习辉拍几千下。但我的教练会反复跟我说，我自己也会反复跟自己说：邓亚萍，如果到了球馆，你就是要马上打500个正手拍！

久而久之，我的大脑，就好像被写入了一个程式，看到球馆，就算身体很累，心情上也不想练习，但我还会很自然地拿起球拍，挥个500下，不挥都不行。

后来，我才知道，这是一个心理学上的专有名词，叫做「执行意图」，英文叫做Implementation Intention。

它的意思是什么呢？用一句话来描述，意思就是：大脑不喜欢你给他问题，大脑喜欢你给他答案。

什么叫大脑不喜欢你给他问题？你理性上想好好学英文，但感性上却又觉得背单字很烦，这对大脑来说，就是一个问题。大脑不喜欢解决问题，最后就是掷色子，有时候理性赢，有时候感性赢。

但如果，在问题发生以前，你曾经给自己，明确设定过一个答案呢？那你的大脑就会很乐意，依照你下过的指令去做。

这种指令，是不需要经过思考，在潜意识中完成的，也就是说，就算你的心理很纠结，你的身体也会自动自发去做的。

你不相信吗？我告诉你，这个心理效应，就是顶尖运动员的秘密。顶尖运动员也是人，重复同样的事情，心里也同样会倦怠的，但顶尖赢家跟普通人的差别是什么呢？赢家靠着「执行意图」，Implementation Intention，还是能坚持下来。

解释完原理，我来跟你说说，具体该怎么做。

简单来说，就是把你要做的事，和一个具体的场景，连结在一起，用一个非常明确的条件句，告诉你的大脑：一但我到了Ｘ场景，我就要做Ｙ事。

比如说，我说过的，一到书桌前，就背20个单字啦，或是：一到公司、坐上工位，就马上和3个客户沟通，甚至是，一起床，就听一则《奥运冠军的心力训练法》，这也可以。

哈哈！聪明的你，肯定能够举一反三，以此类推的。

只是，我要提醒你，「执行意识」这个技巧，也有两个注意事项。

第一个注意事项，这个条件句，不管是场景还是要做的事，都是越明确越好。不管是从我的经验，还是从专业的运动心理学来说，越是明确的场景和明确的待办事项，越是能发挥好效果。

第二个注意事项，「执行意识」这个方法，需要你通过不断的实践，来加强条件反射的程度。尤其在刚开始使用个技巧的时候，要更注意，尽量不要破坏，你自己给自己定的规则，有了Ｘ场景，就一定要做Ｙ事，坐到书桌前，就一定要背20个单词，这样才会加强你下意识的条件反射。

最后，我来总结一下今天的课程。

第一，大脑不喜欢僵局，大脑喜欢你给它具体的指示。

第二，要给大脑指示，可以是如果我到X场景，我就必须做Y事

05、例外控制：如何既保证休息，又保证高效

Posted on 2019-05-07 | Visitors:
Words count in article: 2k | Reading time ≈ 6

印象中的邓亚萍，是个「铁人」。为了训练，可以永远不用休息。这也是中国媒体，对顶尖运动员常用的报道方法。

不把他们当人，而把他们当神。

但第一次和邓亚萍聊天，她就出口反对：「真正有效率的人，都懂得休息、放松」。

所以，我今天想问她的是：世界冠军是如何确保休息和放松，又能同时保持高效率的呢？

今天这八分钟，我想分享的内容，和我的「人设」不太符合。

因为我要来教你，放松和休息的技巧。聊聊我们该怎么做，才能够在保证休息质量的情况下，避免进一步的拖延症。

听到这，一些比较积极的同学，肯定就不乐意了：「亚萍姐！你不是上次才教过我们，怎么像机器人一样，对抗偷懒吗？」

没错，执行力的最高境界，是跟机器人一样，做了计划就能执行。

但就算是机器人，也需要定期检查维修。咱们终究是普通人，有些特别的「例外状况」，适当的放松也是必须的。

我自己学英语的经验，也是如此。

因为我每天都学习超过12小时，有的时候，实在是太累了，这时候硬要爬起来，也是迷迷糊糊，不仅单词背不进去，书读了也记不住。索性还不如多睡半小时，等精神更好一点，才能事半功倍。

读书如此，运动员也是一样。现代的运动科学，会及时的监控运动员的身体，如果肌肉过于疲劳了，教练肯定也会叫停训练，让你休息调整一下再来。

所以咱们今天说的第一件事：千万不要逼着自己一直工作、一直学习。

比偷懒更可怕的，是时间耗费在自我责备中，同时还一事无成。

但是，给自己放松的最大问题，是最怕「开小差变真偷懒」。

原本只是想眯个十分钟，结果最后就睡了一小时。或是睡醒后，就想多玩两下手机。

这种偷懒的状况，是不是你也有经历过？

这种情况，在运动员身上，其实同样常见。我把它叫做「开小差变真偷懒」。

很多人都知道，运动员最害怕的是伤病。

但是，一方面是现在医学有很大进步，可以控制伤病。另一方面，很多时候，身体的状态也不绝对影响表现。

现在NBA雷霆队的保罗乔治 Paul George，也是在伤病后打出生涯最佳的成绩；而我自己呢，更是在伤病缠身的状况下，还拿到1996年的奥运金牌。

所以很多时候，伤病最可怕的，不是伤病本身，而是它会消磨人的意志，让人的精神脱离高强度的竞争状态。

伤病的时候，运动员会突然拥有大量的休息时间。在这个时间里面，啥都不能干。一下子松懈下来，一些人身材就开始走样了，也有一些人，心态就松弛了。

总之，这些人就算伤病痊愈了，也基本废掉了。

换句话说，伤病不可怕，这种「伤病变偷懒」的心态，才是最可怕的。

无论是开小差变真偷懒，还是伤病变偷懒，背后的问题是一样的。

所以重点是，比起那些心态松弛的运动员，为啥有些运动员，能从伤病中振作？

差别在于一个小技巧，这个技巧，叫做「例外控制」。

面对突如其来的休整，松弛下来的球员，真的会像放假一样，原本要用来训练的时间，就用来打游戏、或做与训练无关的事。

但能够从伤病中走出来的球员，都有一个特色，即使是什么都不能干的状态，这些球员，还是会制定严格的「训练日程」。

你打开这个计划，你会看到，上面会有对比赛视频的复盘、会有适当的康复训练。甚至连休息、吃饭甚至睡觉的时间，都规定得一清二楚。

这两种运动员的行动，究竟差在哪？

其实，跟意志力没多大关系。

从心理学上的观点来说，人一但想要开小差，想要休息一下，这种时候，感性的一面，通常就会接管大脑，而一旦感性的一面接管，那理性大脑就没啥事了。

这也就是为什么，那些让感性大脑接管的运动员，会整个垮掉；这也是为什么，明明只是想打盹五分钟，最后就会拖延犯懒，一个小时都给耗没了。

反过来说，为什么厉害的运动员，能够在伤病中，还能有好表现？就是因为，虽然是休整阶段，但我们还是能用理性大脑、甚至是靠潜意识去控制我们自己。

图注：姚明在受伤后，仍然能坚持训练。

这是怎么做到的？

其实方法很简单，就像上一堂课《执行意图》里说的一样：大脑喜欢事先设定的答案，如果你早就已经设定过「该怎么应对例外状况」，那大脑就更有可能乖乖听话。

厉害的运动员，是用详细的计划，控制自己在休整的时间不被偷懒的情绪接管。这就是一种「例外控制」的做法。

所以，你在日常生活中，又可以怎样使用这个技巧呢？

简单来说，就是提前设定一个条件句：「如果发生Ｘ例外情况，那么就应该做Ｙ事。」

这里的Y事，就是根据你的状况，可以被接受的休息计划。

同样以早起背单词为例，你可以提前就跟自己设定：

如果早上真的起不来，那可以允许自己多睡半小时；

如果还是困，那么先起床刷个牙，不用背单词，出去跑步。

如果中间单词背不下来，那么可以听听音乐、打打鸡血。

虽然多睡半小时，或是出去跑步，看起来都是放松，但是比起让感性大脑彻底接管，睡掉一个早上，这已经是相对有效率的方式了。

有了这些设定，就可以控制自己，在遇到例外状况，想要放松的时候，不会一放松就松懈、一放松就崩溃。

因为遇到这些例外状况，首先启动的是你的理性大脑，你马上就能找到事先做好的预案。而不是手足无措，立刻被感性大脑接管。

当然，设定例外状况，也有注意事项。

第一，对应不同的例外状况，采取的行动一定要明确。

比如说，你可以说：早上起不来，那就多睡半小时；但你不能说，早上起不来，就睡到饱为止。

第二，就算有例外状况，整体的目标，也不能有大的变动。

比如说，今天预计要背100个单词，却因为起晚了，只能背80个。那少背的单词呢，就要平摊到接下来三天里，每天多背一些单词。不能让例外状况，干扰到整体进度。

好啦，我为你总结一下今天的课程。

第一，不要逼迫自己一直在工作，主动放松才能更好进步。

第二，休息的时候，「感性大脑」很容易就会接管大脑，让人控制不了惰性。

第三，可以设定一个「如果发生Ｘ例外情况，那么就可以做Ｙ事」的条件句，拒绝感性大脑接管。

我长大一点后才了解到，这句话，真的比我之前的自我激励，要好上太多了。因为这句话的关键，在引导我少想赢的结果一点，多想赢的过程一些。

什么意思？

教练的指示，是要我「跟训练时一样打就好」。他的言下之意，就是要我用训练时的心态，去看待比赛。

问题来了，训练的时候，跟正式比赛的时候，心态有什么不同？

比赛更在乎结果，而训练更在意打好每一颗球。

比赛的时候，失误一分，可能就会决定胜败，一不小心，就会对每颗球斤斤计较；但训练的时候，上一球失误了，没事，这球再补回来就好。

在训练时，我对自己打气的话也会变成：邓亚萍！打好这一球！

如果真的打好了，我更是会大喊「漂亮！」来庆祝、来激励自己。比激情，一点都不比「我最棒」来得差。

那如果打不好、失误了，怎么办？没事，那就打好下一球呗！

在这个过程中，我都不去想，我到底会不会赢，赢了又如何，而是想，我该如何把这一球打好？

而当我把这种心态套用到正式比赛，给自己打鸡血的话，变成「邓亚萍！你的准备很充足！你可以打好这一球！」我发现，我更冷静、更专注，甚至也更兴奋了。

后来，我学习了运动心理学之后，才知道这其中的道理，真的特别简单。

用一句话总结，叫做「要骗过自己的大脑，没那么容易」。

什么意思？

赢不赢，那不是我能控制的。至少在我小的时候，我客观上，未必是最强的那个。如果真是客观上最强，我又需要自我激励干嘛？对于结果，我就是没有把握、不能确定，才要给自己打鸡血，不是吗？

明明没把握，结果我还是说「我是最棒的！我一定能赢！」这种做法，就好像要大脑马上180度转向，那肯定做不到嘛！大脑当然会反弹，反过来质疑「凭什么」呀！

说得更直接一点，如果自我暗示那么容易，那减肥可容易了！你再也不用天天吃沙拉、天天健身，你只要每天早上起来告诉自己，「我最瘦」，不就够了？很明显不是这样嘛！

所以怎么办呢？没办法把大脑转180度，那就先转30度吧！

当我说「我要打好这颗球」，我的脑海里，就是浮现该如何打好这颗球的经验和练习。这些经验和练习，是可控的，是我能确定、有把握的。

注意。这时候，为什么我会相信自己一定能赢？不是因为我打鸡血，说自己是最棒的，而是因为我说服了自己，我能打好每一分球。

越是想赢、越是想要有自信，就越不应该太在意结果，应该多在意「打好每一分球」。

核心的原则，就是针对过程打气，不要给结果打气。

要大考，有个地方觉得还念不熟，你可以这样跟自己说：虽然有地方没读熟，但基本的概念我都掌握了，只要我细心地回答，我可以把基本分都拿到！

其他的状况，你以此类推就好。

只是，我要再一次提醒你，用正确的方法打鸡血，可以激发你的情绪，把准备好的东西，完美的表现出来。但是，如果你不愿意锻炼专业技能，那你不管用什么方式自我激励，也都是白搭的。

是时候，来总结一下啦。

第一，想要自我激励，不要想一步到位。你的大脑不会180度转向，相信你一定能赢。

第二，正确地方法，是关注过程，激励自己把一件件小事做好。一次转30度。

自 Java 9 开始，Jdk 里面为集合（List/ Set/ Map）都添加了 of 和 copyOf 方法，它们两个都用来创建不可变的集合，来看下它们的使用和区别。

示例2：

示例1和2代码差不多，为什么一个为true,一个为false?

来看下它们的源码：

static List of(E… elements) { 

switch (elements.length) { // implicit null check of elements  

    case 0:  

        return ImmutableCollections.emptyList();  

    case 1:  

        return new ImmutableCollections.List12<>(elements[0]);  

    case 2:  

        return new ImmutableCollections.List12<>(elements[0], elements[1]);  

    default:  

        return new ImmutableCollections.ListN<>(elements);  

}

}

static List copyOf(Collection<? extends E> coll) { 

return ImmutableCollections.listCopy(coll);

}

static List listCopy(Collection<? extends E> coll) { 

if (coll instanceof AbstractImmutableList && coll.getClass() != SubList.class) {  

    return (List<E>)coll;  

} else {  

    return (List<E>)List.of(coll.toArray());  

}

}
可以看出 copyOf 方法会先判断来源集合是不是 AbstractImmutableList 类型的，如果是，就直接返回，如果不是，则调用 of 创建一个新的集合。

示例2因为用的 new 创建的集合，不属于不可变 AbstractImmutableList 类的子类，所以 copyOf 方法又创建了一个新的实例，所以为false.

Java 11还加入了一个比较特殊的垃圾回收器——Epsilon，该垃圾收集器被称为“no-op”收集器，将处理内存分配而不实施任何实际的内存回收机制。 也就是说，这是一款不做垃圾回收的垃圾回收器。这个垃圾回收器看起来并没什么用，主要可以用来进行性能测试、内存压力测试等，Epsilon GC可以作为度量其他垃圾回收器性能的对照组。大神Martijn说，Epsilon GC至少能够帮助理解GC的接口，有助于成就一个更加模块化的JVM。

Java 10中增加了本地变量类型推断的特性，可以使用var来定义局部变量。尽管这一特性被很多人诟病，但是并不影响Java继续增强他的用法，在Java 11中，var可以用来作为Lambda表达式的局部变量声明。

移除Java EE和CORBA模块

早在发布Java SE 9的时候，Java就表示过，会在未来版本中将Java EE和CORBA模块移除，而这样举动终于在Java 11中实施。终于去除了Java EE和CORBA模块。

HTTP客户端进一步升级

JDK 9 中就已对 HTTP Client API 进行标准化，然后通过JEP 110，在 JDK 10 中进行了更新。在本次的Java 11的更新列表中，由以JEP 321进行进一步升级。该API通过CompleteableFutures提供非阻塞请求和响应语义，可以联合使用以触发相应的动作。 JDK 11完全重写了该功能。现在，在用户层请求发布者和响应发布者与底层套接字之间追踪数据流更容易了，这降低了复杂性，并最大程度上提高了HTTP / 1和HTTP / 2之间的重用的可能性。

建造者模式
建造者模式（Builder Pattern）使用多个简单的对象一步一步构建成一个复杂的对象。这种类型的设计模式属于创建型模式，它提供了一种创建对象的最佳方式。

一个 Builder 类会一步一步构造最终的对象。该 Builder 类是独立于其他对象的。

介绍
意图：将一个复杂的构建与其表示相分离，使得同样的构建过程可以创建不同的表示。

主要解决：主要解决在软件系统中，有时候面临着”一个复杂对象”的创建工作，其通常由各个部分的子对象用一定的算法构成；由于需求的变化，这个复杂对象的各个部分经常面临着剧烈的变化，但是将它们组合在一起的算法却相对稳定。

何时使用：一些基本部件不会变，而其组合经常变化的时候。

如何解决：将变与不变分离开。

关键代码：建造者：创建和提供实例，导演：管理建造出来的实例的依赖关系。

应用实例： 1、去肯德基，汉堡、可乐、薯条、炸鸡翅等是不变的，而其组合是经常变化的，生成出所谓的”套餐”。 2、JAVA 中的 StringBuilder。

优点： 1、建造者独立，易扩展。 2、便于控制细节风险。

缺点： 1、产品必须有共同点，范围有限制。 2、如内部变化复杂，会有很多的建造类。

使用场景： 1、需要生成的对象具有复杂的内部结构。 2、需要生成的对象内部属性本身相互依赖。

注意事项：与工厂模式的区别是：建造者模式更加关注与零件装配的顺序。

注意：定义中“将一个复杂的构建过程与其表示相分离”，表示并不是由建造者负责一切，而是由监工负责控制（定义）一个复杂的构建过程，由各个不同的建造者分别负责实现构建过程中所用到的所有构建步骤。不然，就无法做到“使得同样的构建过程可以创建不同的表示”这一目标。

建造者角色：

public abstract class Builder {
public abstract void buildPart1();
public abstract void buildPart2();
public abstract void buildPart3();
}
监工角色：

public class Director { // 将一个复杂的构建过程与其表示相分离
private Builder builder; // 针对接口编程，而不是针对实现编程
public Director(Builder builder) {
this.builder = builder;
}
public void setBuilder(Builder builder) {
this.builder = builder;

public void construct() {   // 控制（定义）一个复杂的构建过程
    builder.buildPart1();

        builder.buildPart2();
    }
    builder.buildPart3();

桥接模式
桥接（Bridge）是用于把抽象化与实现化解耦，使得二者可以独立变化。这种类型的设计模式属于结构型模式，它通过提供抽象化和实现化之间的桥接结构，来实现二者的解耦。

这种模式涉及到一个作为桥接的接口，使得实体类的功能独立于接口实现类。这两种类型的类可被结构化改变而互不影响。

我们通过下面的实例来演示桥接模式（Bridge Pattern）的用法。其中，可以使用相同的抽象类方法但是不同的桥接实现类，来画出不同颜色的圆。

介绍
意图：将抽象部分与实现部分分离，使它们都可以独立的变化。

主要解决：在有多种可能会变化的情况下，用继承会造成类爆炸问题，扩展起来不灵活。

何时使用：实现系统可能有多个角度分类，每一种角度都可能变化。

如何解决：把这种多角度分类分离出来，让它们独立变化，减少它们之间耦合。

关键代码：抽象类依赖实现类。

应用实例： 1、猪八戒从天蓬元帅转世投胎到猪，转世投胎的机制将尘世划分为两个等级，即：灵魂和肉体，前者相当于抽象化，后者相当于实现化。生灵通过功能的委派，调用肉体对象的功能，使得生灵可以动态地选择。 2、墙上的开关，可以看到的开关是抽象的，不用管里面具体怎么实现的。

优点： 1、抽象和实现的分离。 2、优秀的扩展能力。 3、实现细节对客户透明。

缺点：桥接模式的引入会增加系统的理解与设计难度，由于聚合关联关系建立在抽象层，要求开发者针对抽象进行设计与编程。

使用场景： 1、如果一个系统需要在构件的抽象化角色和具体化角色之间增加更多的灵活性，避免在两个层次之间建立静态的继承联系，通过桥接模式可以使它们在抽象层建立一个关联关系。 2、对于那些不希望使用继承或因为多层次继承导致系统类的个数急剧增加的系统，桥接模式尤为适用。 3、一个类存在两个独立变化的维度，且这两个维度都需要进行扩展。

注意事项：对于两个独立变化的维度，使用桥接模式再适合不过了。

在代理模式（Proxy Pattern）中，一个类代表另一个类的功能。这种类型的设计模式属于结构型模式。

在代理模式中，我们创建具有现有对象的对象，以便向外界提供功能接口。

介绍
意图：为其他对象提供一种代理以控制对这个对象的访问。

主要解决：在直接访问对象时带来的问题，比如说：要访问的对象在远程的机器上。在面向对象系统中，有些对象由于某些原因（比如对象创建开销很大，或者某些操作需要安全控制，或者需要进程外的访问），直接访问会给使用者或者系统结构带来很多麻烦，我们可以在访问此对象时加上一个对此对象的访问层。

何时使用：想在访问一个类时做一些控制。

如何解决：增加中间层。

关键代码：实现与被代理类组合。

应用实例： 1、Windows 里面的快捷方式。 2、猪八戒去找高翠兰结果是孙悟空变的，可以这样理解：把高翠兰的外貌抽象出来，高翠兰本人和孙悟空都实现了这个接口，猪八戒访问高翠兰的时候看不出来这个是孙悟空，所以说孙悟空是高翠兰代理类。 3、买火车票不一定在火车站买，也可以去代售点。 4、一张支票或银行存单是账户中资金的代理。支票在市场交易中用来代替现金，并提供对签发人账号上资金的控制。 5、spring aop。

优点： 1、职责清晰。 2、高扩展性。 3、智能化。

缺点： 1、由于在客户端和真实主题之间增加了代理对象，因此有些类型的代理模式可能会造成请求的处理速度变慢。 2、实现代理模式需要额外的工作，有些代理模式的实现非常复杂。

使用场景：按职责来划分，通常有以下使用场景： 1、远程代理。 2、虚拟代理。 3、Copy-on-Write 代理。 4、保护（Protect or Access）代理。 5、Cache代理。 6、防火墙（Firewall）代理。 7、同步化（Synchronization）代理。 8、智能引用（Smart Reference）代理。

注意事项： 1、和适配器模式的区别：适配器模式主要改变所考虑对象的接口，而代理模式不能改变所代理类的接口。 2、和装饰器模式的区别：装饰器模式为了增强功能，而代理模式是为了加以控制。

实现

package cn.cpf.pattern.structure.proxy.cglib;

import net.sf.cglib.proxy.Enhancer;
import net.sf.cglib.proxy.MethodInterceptor;
import net.sf.cglib.proxy.MethodProxy;
import java.lang.reflect.Method;

public class CglibProxy implements MethodInterceptor {
private Object target;

public CglibProxy(Object target) {
    this.target = target;
}

@Override
public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {
    System.out.println("###   before invocation");
    Object result = method.invoke(target, objects);
    System.out.println("###   end invocation");
    return result;
}

public static Object getProxy(Object target) {
    Enhancer enhancer = new Enhancer();
    // 设置需要代理的对象
    enhancer.setSuperclass(target.getClass());
    // 设置代理人
    enhancer.setCallback(new CglibProxy(target));
    return enhancer.create();
}

}
